## Workstream A — Architecture Modernization (Week 1)

### A-1: Split api_server.py into Route Modules
- **Current:** 1276-line monolith handling all 19 endpoints
- **Target:** `api_server.py` shrinks to ~200 lines (lifespan, middleware, router mounts)
- **New files:**
  - `routes/__init__.py`
  - `routes/query.py` — `/query`, `/query/stream`
  - `routes/chat.py` — `/chat`, `/chat/stream`, `/chat/{session_id}`
  - `routes/ingest.py` — `/ingest`, `/ingest/all`, `/ingest/status/{job_id}`, `/upload`
  - `routes/agent.py` — `/agent/query`
  - `routes/management.py` — `/papers`, `/papers/{id}`, `/purge/*`, `/stats`, `/cache/*`, `/search`
- **Approach:** Create `APIRouter` per module. Each router owns its Pydantic models. Main app mounts all routers. Shared deps (auth, cache invalidation) move to `deps.py`.
- **Effort:** 1 day

### A-2: Extract LLM Client to Dedicated Module
- **Current:** `rag.py:20-21` uses `__import__('threading')` hack to avoid circular imports
- **Target:** New `llm_client.py` contains pool management, failover logic, streaming generator
- **What moves:** `_client_pool`, `_client_lock`, `_client_index`, `_init_client_pool`, `_next_client_idx`, `_get_client`, `generate_with_failover`, `llm_generate_stream`, `_is_transient`, `_circuit_breaker`
- **What stays in rag.py:** `retrieve_context`, `format_context`, `build_prompt`, `extract_citations`, `answer_question*`, `answer_with_history`, `prepare_*_for_stream`
- **Effort:** 0.5 day

### A-3: SQLite-Backed Session and Job Persistence
- **Current:** Pure Python dicts `_sessions` and `_jobs` vanish on server restart
- **Target:** `aiosqlite` for async persistence, no external dependencies
- **Schema:**
  ```
  sessions: id TEXT PRIMARY KEY, data TEXT (JSON), created_at TEXT, updated_at TEXT
  jobs: id TEXT PRIMARY KEY, data TEXT (JSON), status TEXT, submitted_at TEXT, completed_at TEXT
  ```
- **New file:** `persistence.py` — `SessionStore` and `JobStore` classes wrapping SQLite
- **Migration:** On startup, create tables if missing. In-memory dicts become write-through caches.
- **Effort:** 1 day

### A-4: Request ID Propagation
- **Current:** No correlation between log lines from the same HTTP request
- **Target:** UUID injected per request, available in all log output and response headers
- **Approach:** FastAPI middleware sets `contextvars.ContextVar`. Custom `logging.Filter` reads it. Response includes `X-Request-ID` header.
- **New file:** `middleware.py` — `RequestIdMiddleware` class
- **Effort:** 0.5 day

### A-5: Graceful Shutdown
- **Current:** `start_server.py` uses `subprocess.run()` which doesn't forward SIGTERM
- **Target:** Direct `uvicorn.run()` invocation with shutdown hooks
- **Changes:** `start_server.py` calls `uvicorn.run()` directly. Add cleanup function that flushes in-flight requests, closes ChromaDB client, releases model memory.
- **Effort:** 0.5 day

## Workstream B — Retrieval Quality (Week 2)

### B-1: ColBERT Multi-Vector Retrieval
- **Opportunity:** BGE-M3 produces ColBERT token-level vectors but only the dense embedding is used
- **What to build:** `colbert_index.py` stores per-token vectors alongside ChromaDB. At query time, compute MaxSim between query tokens and document tokens. Fuse dense cosine score + ColBERT score with weighted combination.
- **New files:** `colbert_index.py` — `ColBERTIndex` class with `add()`, `search()`, `save()`, `load()`
- **Changes to:** `rag.py:retrieve_context()` — add ColBERT rerank stage after BM25 fusion
- **Expected gain:** +15-25% recall for Indic language queries where word order varies significantly from English
- **Effort:** 2 days

### B-2: HyDE (Hypothetical Document Embeddings)
- **Concept:** For complex multi-hop queries, generate a hypothetical answer first, embed it, then retrieve against that embedding. The hypothetical answer bridges the lexical gap between query and document.
- **Changes to:** `rag.py:retrieve_context()` — add `use_hyde=True` path
- **Flow:** If enabled, make one short LLM call (`max_output_tokens=256`) to draft a hypothetical answer, embed it with BGE-M3, retrieve top-k using that embedding, then rerank normally.
- **Fallback:** If LLM call fails, silently fall back to standard retrieval
- **Config:** `USE_HYDRIDE = os.getenv("USE_HYDE", "false").lower() == "true"` in `config.py`
- **Effort:** 1 day

### B-3: Automatic Metadata Enrichment on Ingest
- **Current:** Paper metadata comes only from PDF extraction (title from font size, sections from headers)
- **Target:** Auto-fetch authors, year, DOI, citation count from arXiv/OpenAlex after PDF processing
- **Approach:** In `ingest.py:ingest_pdf()`, after `process_pdf()`, query arXiv by title fuzzy match. If match found within confidence threshold, merge enriched metadata into chunk metadata before embedding.
- **New file:** `metadata_enrich.py` — `enrich_from_arxiv(title) -> dict | None`
- **Config:** `ENRICH_METADATA = os.getenv("ENRICH_METADATA", "true").lower() == "true"`
- **Effort:** 1 day

### B-4: Cross-Ingestion Paper Deduplication
- **Current:** `calculate_sha256()` deduplicates same-file re-uploads. Different filenames of the same paper create duplicate chunks.
- **Target:** Before embedding, check if a paper with near-identical title already exists
- **Approach:** In `ingest_paper()`, query ChromaDB for existing papers, compute title similarity (token overlap or Levenshtein). Skip if similarity > 0.9 and same year. Log the skip with reason.
- **New function:** `vector_store.find_similar_paper(title, threshold=0.9) -> str | None` returns matching paper_id or None
- **Effort:** 0.5 day

## Workstream C — User Experience (Week 3)

### C-1: Frontend SSE Consumer for Streaming
- **Current:** Backend streaming is ready but `static/index.html` still uses `fetch()` for `/query` and `/chat`
- **Target:** Switch to SSE consumption so users see tokens appear in real-time
- **Approach:** Replace `fetch()` with `fetch()` + `ReadableStream` reader. Parse `data:` lines. Append `chunk` events to answer div. Handle `done` event for citations. Fall back to non-streaming if SSE fails.
- **Changes to:** `static/index.html` — JavaScript section only, ~50 lines changed
- **Effort:** 1 day

### C-2: Query Feedback Loop
- **Concept:** After each answer, show thumbs up/down buttons. Store feedback alongside query, retrieved chunks, and generated answer for future analysis.
- **New endpoint:** `POST /feedback` with body `{query_id, rating: "up"|"down", comment?: str}`
- **Storage:** SQLite table `feedback(id TEXT PK, query_id TEXT, rating TEXT, comment TEXT, created_at TEXT)`
- **Frontend:** Add feedback buttons below each answer in the chat UI
- **Benefit:** Enables A/B testing of retrieval strategies and identification of weak corpus areas
- **Effort:** 1 day

### C-3: BibTeX Export for Search Results
- **New endpoint:** `GET /search/export?format=bibtex&query=...&top_k=10`
- **Approach:** Reuse existing retrieval pipeline, format results as BibTeX entries using title, authors, year, DOI from metadata
- **Changes to:** `routes/management.py` (new router) or `routes/query.py`
- **Effort:** 0.5 day

### C-4: Conversation Memory Across Sessions
- **Current:** Sessions expire after `SESSION_MAX_AGE_HOURS`. No persistent user preferences.
- **Target:** Store per-user preferences (preferred language, topic interests, strategy preference) that persist across sessions
- **Schema:** `user_prefs(user_id TEXT PK, prefs TEXT (JSON), updated_at TEXT)`
- **Integration:** `query_planner_node` injects user preferences into the planning prompt. Strategy selection defaults to user's saved preference.
- **Config:** `ENABLE_USER_PREFS = os.getenv("ENABLE_USER_PREFS", "false").lower() == "true"`
- **Effort:** 1 day

## Risk Register

| Risk | Mitigation |
|------|------------|
| SQLite under concurrent load | Use WAL mode + connection pooling. Read-heavy workload. |
| ColBERT memory overhead | Store token vectors as float16. Cap at 512 tokens per document. |
| arXiv rate limiting during bulk ingest | Add 1s delay between enrichment calls. Cache results. |
| Frontend SSE compatibility | Fallback to non-streaming fetch if EventSource unavailable. |
| Circular imports after LLM extraction | New module depends on nothing in rag.py. rag.py imports llm_client. |
